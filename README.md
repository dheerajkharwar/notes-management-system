# AI Notes Management System

Laravel 12 notes application for the PHP Backend + AI Usage assignment. It includes JSON CRUD APIs, pagination, semantic search, an OpenAI-powered summary endpoint, a simple frontend UI, tests, and OpenAPI documentation.

## Features

- Create, update, delete, view, and list notes through JSON APIs.
- Paginated note listing with `page` and `limit` query parameters.
- Semantic search using OpenAI embeddings when configured, with deterministic local embeddings as a fallback.
- Summary generation endpoint using the OpenAI Responses API when configured, with a local extractive fallback.
- Frontend workspace built with Blade, Vite, Tailwind CSS, and vanilla JavaScript.
- API validation, route throttling, Eloquent-based database access, and clean JSON resources.
- Feature tests for CRUD, validation, search, and summary behavior.

## Setup

```bash
composer install
npm install
cp .env.example .env
php artisan key:generate
```

Configure MySQL or PostgreSQL in `.env`:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=notes_management_system
DB_USERNAME=root
DB_PASSWORD=your_password
```

Then run:

```bash
php artisan migrate
npm run build
php artisan serve
```

Open `http://127.0.0.1:8000`.

## OpenAI API Setup

Add your OpenAI API key to `.env`:

```env
OPENAI_API_KEY=sk-your-key
OPENAI_CHAT_MODEL=gpt-4.1-mini
OPENAI_EMBEDDING_MODEL=text-embedding-3-small
```

When `OPENAI_API_KEY` is set:

- New or updated notes store embeddings from `POST https://api.openai.com/v1/embeddings`.
- `POST /api/notes/{id}/summary` generates summaries through `POST https://api.openai.com/v1/responses`.

When the key is missing or an API request fails, the app falls back to local embeddings and local extractive summaries so demos and tests still work.

## API Documentation

OpenAPI spec: [docs/openapi.yaml](docs/openapi.yaml)

### List Notes

```http
GET /api/notes?page=1&limit=10
```

Returns Laravel pagination metadata and note resources.

### Create Note

```http
POST /api/notes
Content-Type: application/json

{
  "title": "Release checklist",
  "content": "Confirm migrations, test API endpoints, and prepare demo notes."
}
```

Returns `201 Created`.

### Update Note

```http
PUT /api/notes/{id}
Content-Type: application/json

{
  "title": "Updated release checklist",
  "content": "Confirm migrations, tests, API endpoints, and demo notes."
}
```

### Delete Note

```http
DELETE /api/notes/{id}
```

Returns `204 No Content`.

### Get Single Note

```http
GET /api/notes/{id}
```

### Semantic Search

```http
GET /api/notes/search?q=vector%20embeddings&limit=10
```

Returns ranked matches:

```json
{
  "data": [
    {
      "score": 0.72,
      "note": {
        "id": 1,
        "title": "Machine learning roadmap"
      }
    }
  ],
  "meta": {
    "query": "vector embeddings",
    "count": 1,
    "method": "local-hashed-embedding"
  }
}
```

### Generate Summary

```http
POST /api/notes/{id}/summary
```

Returns the note with `summary` populated.

## Database Schema

`notes`

| Column | Type | Notes |
| --- | --- | --- |
| `id` | bigint | Primary key |
| `title` | string | Required, max 180 characters |
| `content` | long text | Required |
| `summary` | text nullable | Generated by OpenAI or local summary endpoint |
| `embedding` | json nullable | OpenAI or local vector representation for semantic search |
| `created_at` | timestamp | Indexed |
| `updated_at` | timestamp | Indexed |

Laravel default `users`, `cache`, and `jobs` tables are also present.

## Architecture

- `App\Http\Controllers\Api\NoteController` handles API endpoints and delegates AI-like work to services.
- `StoreNoteRequest` and `UpdateNoteRequest` centralize validation rules.
- `NoteResource` keeps response shape consistent and hides implementation fields such as embeddings.
- `OpenAIClient` calls OpenAI embeddings and Responses endpoints through Laravel's HTTP client.
- `TextEmbeddingService` requests OpenAI embeddings when configured, otherwise builds a normalized local vector, and computes cosine similarity.
- `NoteSummaryService` requests OpenAI summaries when configured, otherwise ranks note sentences using extracted keywords.
- `resources/views/notes/*` provides separate Blade pages for list, create, view, and edit workflows.

## Security Notes

- SQL injection prevention: all database access uses Eloquent/query builder.
- API validation: Form Requests validate create/update payloads; search and pagination validate query parameters.
- Rate limiting: API routes use `throttle:60,1`.
- Secure API handling: no secrets are exposed in JSON responses; embeddings are stored internally and omitted from resources.
